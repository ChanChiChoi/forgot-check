# 打卡提醒 App — 项目规划

## 项目概述

基于 GPS 地理围栏的打卡提醒应用。当用户靠近或远离指定坐标一定距离时，通过弹窗或通知提醒打卡。防止上下班忘记打卡。

---

## 需求清单

| # | 需求 | 说明 |
|---|------|------|
| 1 | **添加打卡地点** | 用户可在地图上或当前GPS位置添加打卡点，设置名称（如"公司"）和提醒半径（米） |
| 2 | **上班提醒（进入区域）** | 当用户进入指定打卡点半径范围内时，弹出提醒/闹铃提醒打卡 |
| 3 | **下班提醒（离开区域）** | 当用户离开指定打卡点半径范围后，再次提醒打卡 |
| 4 | **位置监控后台运行** | 即使App关闭，后台也能持续监控位置变化 |
| 5 | **管理打卡地点列表** | 可增删改查多个打卡地点，每个地点独立设置半径和提醒模式 |

---

## 功能清单

| # | 功能 | 技术实现 |
|---|------|----------|
| 1.1 | 首页显示打卡地点列表 | `RecyclerView` 列表，每个卡片显示地点名称、坐标、半径、状态 |
| 1.2 | 添加打卡地点对话框 | 弹窗输入名称、半径，自动获取当前GPS坐标，也支持手动输入经纬度 |
| 1.3 | GPS定位获取 | 使用 `FusedLocationProviderClient`（Google Play Services）或系统 `LocationManager` |
| 2.1 | 后台位置监控服务 | `ForegroundService` + 定期位置轮询，避免被系统杀死 |
| 2.2 | 地理围栏判断 | Haversine 公式计算两点距离，判断进出区域 |
| 2.3 | 状态记录 | 记录每个地点的"已进入/已离开"状态，防止重复弹窗 |
| 2.4 | 上班提醒逻辑 | 进入半径范围内 → 触发弹窗提醒"请打卡" |
| 2.5 | 下班提醒逻辑 | 离开半径范围后 → 触发弹窗提醒"请打卡" |
| 3.1 | 弹窗提醒 | 当App在前台时，`AlertDialog` 弹窗 |
| 3.2 | 通知栏提醒 | 当App在后台时，发送系统 `Notification`，点击可打开App |
| 3.3 | 闹铃声音提醒 | 可选配置，提醒时播放闹铃声 |
| 4.1 | 权限申请 | 位置权限（`ACCESS_FINE_LOCATION`）、后台位置权限、通知权限 |
| 4.2 | 后台服务自启动 | 监听 `BOOT_COMPLETED`，手机重启后自动启动监控服务 |
| 5.1 | 删除打卡地点 | 长按或滑动删除 |
| 5.2 | 编辑打卡地点 | 修改名称、半径、坐标 |
| 5.3 | 启用/禁用开关 | 每个地点可单独开关，不删除但暂停监控 |
| 5.4 | 本地数据存储 | `SQLite` 或 `SharedPreferences` 持久化打卡地点数据 |

---

## 技术要点

- **最低 SDK 21**，后台服务需处理 Android 8.0+ 的限制（使用 Foreground Service）
- **定位方案**：优先 `FusedLocationProviderClient`（更省电、更准），fallback 到 `LocationManager`
- **距离计算**：Haversine 公式或 `Location.distanceTo()`
- **数据持久化**：`Room` 数据库或轻量级 `SharedPreferences`（数据量小的话）
- **状态机**：每个打卡地点记录 `inside`/`outside` 状态，避免重复提醒

---

## 权限清单

| 权限 | 用途 |
|------|------|
| `ACCESS_FINE_LOCATION` | 精确GPS定位 |
| `ACCESS_COARSE_LOCATION` | 粗略定位（fallback） |
| `ACCESS_BACKGROUND_LOCATION` | 后台位置监控（Android 10+） |
| `POST_NOTIFICATIONS` | 发送通知栏提醒（Android 13+） |
| `FOREGROUND_SERVICE` | 前台位置监控服务 |
| `FOREGROUND_SERVICE_LOCATION` | 前台服务使用位置信息 |
| `RECEIVE_BOOT_COMPLETED` | 开机自启动监控服务 |
